什么是动态规划

　　动态规划是一种自顶向上的思考方式。什么是自顶向上，我们拿最简单的求斐波那契数列来举例说明。

// 常见的递归
function fn(n) {
  if(n === 0) {
    return 0;
  }
  if(n === 1 || n === 2) {
    return 1;
  }
  return fn(n - 1) + fn(n -2)
}

　　纯递归的时间复杂度是O(n²),可以看到其中有很多重复的计算，虽然可以加入备忘录的模式减少重复计算，时间复杂度为O(N)。这个时候其实已经和动态规划差不多了。但是实现上还是有些冗余的。我们是通过一种递归树的概念，自顶向下思考。而动态规划是反向思考，自底向上思考的。
　　那么什么是自底向上思考呢？我们知道斐波那契数列思考方式是，求f(n)就是求f(n-1) + fn(n-2)的过程。那么当n === 1 || n === 2的时候，我们是知道它的值的。这个时候我们反向思考，f(3)是什么值呢？答案显而易见就是2。由此，我们可以推断出，当 f(1) = 1, f(2) = 2,后面的值都是前两项的和。这种思考方式就是自底向上的。

// 动态规划
function (n) {
  if(n === 0) return 0;
  if(n === 1 || n === 2) return 1;
  let dp = new Array(n + 1).fill(0);
  for(let i = 3; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
    return dp[n];
  }
}

62.不同路径

　　再举例说明一道中等难度题目。 

/* 一个机器人位于一个 m x n 网格的左上角 （起始点在下图中标记为 “Start” ）。
   机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。
   问总共有多少条不同的路径？*/
// 1. 题目中提到只能向下，或者向右。
// 2. 我们直接从终点开始看，从尾部思考问题，终点上面那一格子，只有一种走法，终点左边那个格子也只有一种解法
// 3. 如果我们把这个格子看成一个二维数组，那么可以得出array[i][j] = array[i-1][j] + array[i][j-1]
// 4. 于是有了下面的解法
var uniquePaths = function(m, n) {
  if(m === 1 && n === 1) {
    return 1
  }
 let array = [];
 for(let i = 0; i < n; i++) {
  array[i] = [];
}
 for(let i = 0; i < n; i++) {
   for(let j = 0; j < m; j++){
     if((i == 0 && j !== 0 )|| (j == 0 && i !== 0)) {
       array[i][j] = 1;
     } else if(i == 0 && j == 0){
       array[i][j] = 0;
     } else {
      array[i][j] = array[i-1][j] + array[i][j-1]
     }
   }
 }
 return array[n-1][m-1];
};
