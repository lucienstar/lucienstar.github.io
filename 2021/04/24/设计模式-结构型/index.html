适配器模式


　　适配器模式是一种结构型设计模式， 它能使接口不兼容的对象能够相互合作。

axios

　　在axios里，可以很明显的看到这种适配模式。axios里最终请求都是到request方法。然后返回一个Promise。

Axios.prototype.request = function request(config) {
  ...
  try {
    promise = dispatchRequest(newConfig);
  } catch (error) {
    return Promise.reject(error);
  }
}

　　最终，都是通过dispatchRequest来发送请求。

var adapter = config.adapter || defaults.adapter;
module.exports = function dispatchRequest(config) {
  return adapter(config).then(function onAdapterResolution(response) {
    throwIfCancellationRequested(config);
    // Transform response data
    response.data = transformData(
      response.data,
      response.headers,
      config.transformResponse
    );
    return response;
  }, function onAdapterRejection(reason) {
    if (!isCancel(reason)) {
      throwIfCancellationRequested(config);

      // Transform response data
      if (reason && reason.response) {
        reason.response.data = transformData(
          reason.response.data,
          reason.response.headers,
          config.transformResponse
        );
      }
    }

    return Promise.reject(reason);
  });
}

　　如果没有在config里传入自定义的适配器，就使用默认的适配器。默认的配置器会去判断XMLHttpRequest是否存在，存在就使用xhr,否则使用http这个库。来磨平浏览器和Noded=端的差异。所以axios浏览器端是只支持xhr的吗😐。 

// defaults.js
function getDefaultAdapter() {
  var adapter;
  if (typeof XMLHttpRequest !== 'undefined') {
    // For browsers use XHR adapter
    adapter = require('./adapters/xhr');
  } else if (typeof process !== 'undefined' && Object.prototype.toString.call(process) === '[object process]') {
    // For node use HTTP adapter
    adapter = require('./adapters/http');
  }
  return adapter;
}

理解　　

　　适配器对外暴露接口一致，允许内部有不同的逻辑

组合模式


　　组合模式就比较有意思了，在很多业务场景其实都可以用到，比如统计城市，统计人口，还有菜单组件。树状结构的数据都可以考虑组合模式。对外暴露一致的接口。

// 定义接口规范
interface Compose {
    name: string,
    add(file: CFile): void,
    scan(): void
}

// 树对象 - 文件目录
class CFolder implements Compose {
    fileList = [];
    name: string;

    constructor(name: string) {
        this.name = name;
    }

    add(file: CFile) {
        this.fileList.push(file);
    }

    scan() {
        for (let file of this.fileList) {
            file.scan();
        }
    }
}

// 叶对象 - 文件
class CFile implements Compose {
    name: string;

    constructor(name: string) {
        this.name = name;
    }

    add(file: CFile) {
        throw new Error('文件下面不能再添加文件');
    }

    scan() {
        console.log(`开始扫描：${this.name}`)
    }
}

let mediaFolder = new CFolder('娱乐');
let movieFolder = new CFolder('电影');
let musicFolder = new CFolder('音乐');

let file1 = new CFile('钢铁侠.mp4');
let file2 = new CFile('再谈记忆.mp3');
movieFolder.add(file1);
musicFolder.add(file2);
mediaFolder.add(movieFolder);
mediaFolder.add(musicFolder);
mediaFolder.scan();

装饰器模式


装饰模式是一种结构型设计模式， 允许你通过将对象放入包含行为的特殊封装对象中来为原对象绑定新的行为。

ES7

　　ES7支持了装饰器语法。

@decorator
class Cat {}

class Dog {
    @decorator
    run() {}
}

　　和class一样，只是语法糖。通过Object.defineProperty方法，在类或者方法上加以“装饰“。所以会有一些Babel的插件方便在低版本浏览器支持这种语法。（所以如果不支持Object.defineProperty那就没办法了。😭可恶的我还要考虑IE8）

理解

　　装饰器这种元编程语法，还是挺好使的。
